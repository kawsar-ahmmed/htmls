🔄8. Daily Operator Routine
L1 = no auto-execute (10–15 min morning)
Step 1
Check 3 campaigns spend vs cap (HC-1: ৳3k/day max Phase 1)
Step 2
Check funnel events: Acquisition CTR/Reg rate, Trial Reg→StartTrial, Conversion StartTrial→Purchase
Step 3
If RED on any funnel → PAUSE that campaign, do not "fix by adding budget"
Step 4
If Acquisition CTR fine but Reg rate <2% → landing problem, not ad problem
Step 5
Pixel phase check: CompleteRegistration events last 7d. <50 = still Phase 1. ≥50 = promote Phase 2.
Every Friday:
  • Refresh 1–2 cold creatives if frequency >3.5
  • Review if ৳3k/day still appropriate (BUD-FUN-9 — never spend >30% of available cash)
🚪9. Gate to L2
Promote when ALL true for 7+ days
  • ✓
    Pixel Phase 2
    50–200 CompleteRegistration events trusted
  • ✓
    At least 1 paying customer
    Validates Conversion path end-to-end
  • ✓
    Conversion CR measurable
    StartTrial → Purchase rate, even if low
  • ✓
    Event quality clean
    No duplicates, no broken pages (operator confirms)
Promotion to L2 unlocks: paid Activation funnel, defensive auto-execute, segmented Recovery, fortnightly creative refresh.
🚫10. Hard "Don't Do" List
Anti-patterns from FUN rules
  • ✗
    Run paid Activation campaigns at L1 — burns budget on weak signal
  • ✗
    Set fraud_panel_viewed as Meta optimization event (FUN-10)
  • ✗
    Send recovery audience to /signup or trial page (CR-FUN-8)
  • ✗
    Cold ad → /pricing landing (FUN-4 alignment break)
  • ✗
    Increase Acquisition budget when Conversion is empty/red (FUN-9)
  • ✗
    Skip Recovery cap "because audience is hot" (caps are non-negotiable)
  • ✗
    Pure-discount Recovery ad (CR-FUN-9)
  • ✗
    Spend >30% of available cash, ever (BUD-FUN-9)
  • ✗
    Mix legacy plans (starter/business/premium) into ads — promote v2_monthly + v2_yearly only
